Como subir una WEB a un Hosting y ponerle un nuevo Dominio

En esta pagina te enseño todo lo que necesitas saber, para dejar tu web en internet, luego de diseñarla de manera local, esto lo podes hacer totalmente de manera gratuita, no necesitas pagar nada, pero si queres algo profesional a futuro, es recomendable invertir en un buen hosting y dominio - Escrito por Jhonnyy Dev

Pasos a realizar para lograrlo

  1. Buscar un Hosting
  2. Conseguir un dominio
  3. Definir que tipo de web se va a subir
  4. Configurar base de datos en el hosting
  5. Subir web al hosting
  6. Configurar SSL

1 - Buscar un Hosting

Hosting podemos encontrar de manera gratuita y paga, la ventaja de pagar un hosting, es que dara menos complicaciones para dejar nuestra web funcional, sera en su mayoria mas accesible configurar todo y dispondra de una mayor velocidad de carga nuestro sitio web y seguramente incluyendo certificado SSL completo y dominio, pero en el caso que no quisieramos pagar un hosting, tenemos alternativas gratuitas, como lo son por ejemplo https://www.infinityfree.com/ (recomendado) y https://googiehost.com/es/ entre otros miles que hay.

2 - Conseguir un dominio

El dominio, es parte de la URL del sitio, el nombre y el final despues de poner HTTP://"ESTO.COM", si queremos uno mas personalizado vamos a tener que pagarlo, generalmente tienen duracion de 1 año aproximadamente, tienen mas valor si terminan en .com .net o son palabras que son muy buscadas. Generalmente al contratar un hosting pago, viene en la mayoria de los casos, 1 dominio gratis incluido (en realidad no es gratis, ya estas pagando el hosting, y seguro que te estan cobrando de más porque te dan el dominio), pero sino queres pagar nada, tambien podemos optar por tener alguno gratuito, que vienen incluido en los hosting gratuitos, que anteriormente te mencione, obviamente van a ser dominios largos y poco amigables para escribirlos, pero lo importante es que son gratis y no vas a tener que pagar nada.

3 - Definir que tipo de web se va a subir

Podemos subir una web hecha con Codigo, la cual sera en parte mas sencilla de subir, o por otro lado podemos subir una web realizada con un CMS (Como por ejemplo Wordpress), y subir una web de este estilo, es un poco mas complicado, ya que suelen ser un conjunto mas amplio de archivos, ademas de que hay diversas formas de realizarlo.

4 - Configurar base de datos en el hosting

Esto se debe realizar dependiendo del sitio web que vayamos a subir al hosting, si es una web "Estatica" hecha con codigo, en donde no es necesario almacenar datos de usuario y contraseña por ejemplo, no hace falta configurar nada de esto. en cambio si necesitamos realizar lo anterior, si se debe hacer. Y en el caso que se suba una web hecha con un CMS (como wordpress), tambien debemos configurar una base de datos.
Esto se hace generalmente en la opcion "PhPMyAdmin / base de datos / database, etc" que encontraremos en la web de nuestro hosting. Dentro de la misma buscaremos la de "crear una base de datos" a la misma le asignamos un "nombre, usuario y contraseña", que debemos recordar para acceder a la misma y configurarla en un futuro.

Dependiendo de lo que vayamos a subir al hosting, esto se realiza antes del paso 5 o en algunos casos es indistinto.
Se realiza antes si: Creamos una web con CMS y usamos un plugin como por ejemplo duplicator
Es indistinto si: Subimos una web con CMS y pasamos los archivos mediante Fillezila
Es indistinto si: Subimos una web hecha con codigo.

En el caso de que sea antes la creacion, para el caso de usar duplicator, solo lo unico que tenemos que hacer es crear la base de datos que se explico anteriormente. y en el caso que sea indistinto, se debe realizar una copia de la base de datos en un archivo SQL, e importarla dentro de "PhPMyAdmin o la base de datos que tenga nuestro hosting" una vez que se haya subido la base de datos, tenemos que ir y cambiar la URL que teniamos en local, por la nueva URL que tengamos en el hosting. en el caso de usar CMS de Wordpress la tabla se encuentra en WP_options.

5 - Subir web al hosting

En este punto, hay que tener en cuenta los 2 tipos de páginas que vamos a subir, ya que en cada caso el proceso es distinto

  • Con codigo (html,css + JavaScript, PhP, Typescript, etc)
  • Con CMS (Wordpress, Shopify, etc)

Con codigo (html,css + JavaScript, PhP, Typescript, etc)

Si decidimos subir una web de este formato, sera tan simple como subir archivos a una nube, es cuestion de buscar la carpeta "File Manager" o similar de nuestro hosting, y ubicar los archivos html,css,imagenes, etc. En donde nos indique el hosting.

Con CMS (Wordpress, Shopify, etc)

Si nuestra web es de este estilo, migrarla a un hosting es un poco mas complejo, tenemos diversos metodos
Metodo 1: Habra que descargarse un programa llamado File Zilla (Cliente), el cual basicamente se conecta a la nube del hosting, y nos permite transferir una gran cantidad de archivos, para eso el programa nos va a solicitar 4 datos "host" "usuario" "contraseña" "puerto" y todo esto lo vamos a encontrar, en el mismo hosting en algun apartado que diga FTP, el programa es bastante sencillo, nos muestra solamente nuestros archivos y carpetas, como lo veriamos en cualquier ventana. Una vez que estemos conectados, es arrastrar archivos de un lado a otro, de nuestra PC al Hosting. Hay que tener en cuenta donde pasar los archivos, generalmente el hosting nos indica donde debemos hacerlo, ya sea entrando a la pagina web, por primera vez, sin configurar nada, desde 0 ni bien nos da el dominio o en algunos casos revisando el File Manager o "nube similar" leyendo lo que nos pueda indicar. Despues tendremos que tener la base de datos creada, esto lo hacemos en donde tengamos el apartado "databases, mysql, base de datos, etc. en nuestro hosting, una vez creada ingresamos al PhPMyAdmino el que disponga el hosting, y buscamos la opcion de importar, ahi debemos subir nuestro archivo SQL de nuestra base de datos previamente exportado de nuestro sitio web creado en wordpress en local, y finalizando con la base de datos, debemos ubicar el archivo wp_config.php (que estara con los archivos que hemos subido) y editarlo con los datos de nuestra nueva base de datos (nombre de base de datos, usuario, contraseña, etc) sino hacemos esto, no vamos a poder cargar nuestro sitio. Luego de realizar lo anterior, nuestro sitio Web, ya deberia por lo menos ingresarse. Pero no todo termina ahi, porque es muy probable que imagenes no esten mostrandoce o algun que otro contenido faltante, y esto se arregla instalando un plugin llamado "Better Search Replace" el cual te pide que pongas, el dominio que usabas en local y el nuevo que conseguiste en el hosting.
Metodo 2: En el caso de Wordpress, se puede descargar un plugin llamado “Duplicator”, el cual debemos usarlo primero en Local, para hacer un respaldo de nuestro sitio. Este Plugin va a crear 2 archivos, 1 es la web y el otro es un instalador de la misma. Estos 2 archivos debemos subirlos a la nube de nuestro hosting. Luego para empezar a correr el instalador, se debe ingresar al mismo mediante una URL que tenga la ubicacion exacta del mismo, por ejemplo "www.tu-sitio.com/installer.php", previamente recordar que debe estar creada la base de datos en el hosting, ya que al empezar el instalador nos solicita, nombre, usuario y contraseña de la misma. Luego que termina, el sitio web, ya estaría visible.

6 - Configurar SSL

Con este último paso podemos dejar nuestra web funcional al 100%, si bien llegando al paso 5, ya es visible. Eso no quiere decir que esté funcionando bien. Ya que debemos instalar algo que se llama certificado SSL, que es básicamente demostrar que nuestra web es segura, y el inicio de la URL cambia de “http” a “https”

Para instalar el SSL va a ser más fácil y “completo” si nuestro hosting es pagado, ya que seguramente tengamos una opción dentro de nuestro hosting que lo activa de manera sencilla, sin muchas complicaciones. Y en el caso que nuestro hosting sea gratuito, tendrá un poco más de complicaciones instalarlo y no sea “completo“, pero dentro de todo funcional casi al 100%

Ahora sin más rellenó, para instalarlo de forma gratuita, vamos a algún apartado que diga SSL en nuestro hosting. Y le damos a alguna opción que diga de solicitar SSL, puede pasar que haya milagrosamente algo que lo haga automático y sino, buscar la opción de solicitar un CSR, nos dará un código que debemos introducir en algún servicio de SSL externo gratuito, como por ejemplo ZeroSSL, al poner el código de solicitud de certificado, nos darán 3 archivos, que son útiles para activar el SSL en nuestro hosting. 1 la clave privada, 2 el certificado y 3 el certificado bundle o cadena, este último, no lo vamos a poder instalar, en un hosting gratuito y eso puede provocar que nuestro sitio web no funcione en todos los navegadores, aunque serían la minoría el 1% de 99% que si estarían funcionando. Estos 3 archivos contienen un código, que debemos visualizar abriéndolos como bloc de notas, copiarlos y pegarlos en donde nos sea solicitado en el hosting, luego de eso. Suelen pedir que subamos un archivo en una ubicación específica, en la nube del hosting y corroborar que se puede acceder a él, mediante una URL, si eso funciona en breves minutos, debería quedar instalado. Y veremos nuestra URL con https y un candado que indica que nuestro sitio es seguro.

Pero no todo termina en lo anterior, faltan algunas cosillas más, hay que cambiar en la base de datos (si es que tenemos) la URL, ya que seguramente debe figurar con HTTP y debemos cambiar a HTTPS. Y si nuestra página está hecha con Wordpress, debemos ingresar al escritorio de Wordpress e ir a los ajustes y cambiar la URL si figura con HTTP a HTTPS. Y finalmente ahora si, la web estaría funcionando al 100%. Felicitaciones si llegaste hasta este punto!

Errores que se me presentaron y cómo solucionarlos

Bueno vamos a hablar de las complicaciones que tuve al subir mi sitio web creado con Wordpress, a un hosting gratuito, especificamente el InfinityFree y las soluciones que encontre para resolver estos problemas

  1. Si se instala un certificado SSL gratuito puede haber complicaciones con algunos navegadores, ya que no lo detectan como si fuera de confianza, eso porque lo descriminan por ser gratis, se soluciona, buscando otro y borrando el que se haya instalado, no puede haber 2 en simultaneo para la misma URL, sino el sitio web no funciona
